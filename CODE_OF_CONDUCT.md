# Code of Conduct

## Our Commitment

We want NORMALIZING to be a project where people can collaborate without
needless friction, disrespect, or intimidation. Contributors, maintainers, and
users are expected to treat each other with patience, clarity, and good faith.

## Expected Behavior

- Be respectful in code reviews, issues, discussions, and pull requests.
- Assume good intent, especially when someone is still learning the codebase.
- Give technical feedback that is specific, calm, and actionable.
- Keep disagreement about code focused on behavior, risk, maintainability, and
  product impact.
- Respect different experience levels, backgrounds, and communication styles.

## Unacceptable Behavior

- Harassment, threats, hate speech, or discriminatory language
- Personal attacks, insults, or repeated hostile behavior
- Deliberate derailment, dogpiling, or bad-faith reporting
- Publishing private information without consent
- Any conduct that makes collaboration feel unsafe or exclusionary

## Scope

This policy applies within project spaces, including:

- Git commits, pull requests, issues, and code reviews
- Project discussions and documentation
- Direct communication conducted in the context of project work

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, or
other contributions that do not align with this Code of Conduct.

Depending on severity, maintainers may:

- ask for clarification or a reset in tone
- issue a warning
- temporarily limit participation
- remove a contributor from project spaces

## Reporting

If you experience or witness behavior that violates this policy, report it to
the maintainers through the repository contact path or the security contact
listed in `SECURITY.md` when privacy is important.

Reports should include:

- what happened
- where it happened
- who was involved
- any relevant screenshots, logs, or links

We will review reports in good faith and try to respond promptly and fairly.
